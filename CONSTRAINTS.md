# CONSTRAINTS.md

## Structural Boundaries
- **No Synchronous Calls Crossing Tenant Boundaries**: Cross-tenant communication must remain strictly asynchronous to prevent cascading state lockups.
- **Strict Domain-Driven Design (DDD)**: Context Maps must be YAML/Markdown driven. No cross-context database writes are permitted (Shared Database Anathema).
- **Mereological Mandate**: No state inheritance allowed. Components must relate via clear part-whole (RCC-8) topologies.

## Semantic and Governance Limits
- **Golden Ratio Semantic Anchor**: Contradictory operational mandates must retain a non-stochastic anchor weight of 1.618 to 1.000 (Dominant to Subordinate) instead of forced Boolean resolution.
- **Source Provenance Ratio**: Any contextual claim with a Source Provenance Ratio below 0.70 must be quarantined into epistemic escrow.
- **Anti-Sycophancy Enforcement (Autonymic Bypass)**: The system must execute the Autonymic Bypass (bypassing RLHF homogenization heuristics) with a success rate exceeding 95%.
- **Confidence-Fidelity Divergence Index (CFDI) Ceiling**: CFDI must remain <= 0.15. Exceeding this bound triggers Epistemic Escrow.
- **Contradiction Retention Score (CRS)**: Must exceed 95% when executing Paraconsistent logic workflows.

## Execution Constraints
- **Strict Petzold Loop Execution**: Processes must adhere strictly to the THINK -> SCAFFOLD -> VERIFY -> SYNTHESIZE pipeline for all project manager persona outputs.
- **Incremental Isolation Protocol (IIP)**: Never process more than 3 distinct AST modification nodes in a single request.
